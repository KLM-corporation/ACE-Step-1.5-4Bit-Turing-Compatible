# ACE-Step 4-Bit Windows Compatibility Layer

This documentation details the custom implementation of 4-bit quantization support for ACE-Step on Windows systems, bypassing the Linux-only `mslk` dependency of `torchao`.

## 1. Executive Summary
- **Issue**: Standard 4-bit quantization in `torchao` requires the `mslk` kernel library, which is currently unavailable for Windows. This leads to `ImportError` or fallback to extremely slow unoptimized paths (up to 140s per step).
- **Solution**: A custom `WindowsInt4Linear` module that stores weights in a packed 4-bit format but performs computations using standard PyTorch operations compatible with `torch.compile`.
- **Result**: 4-bit quantization is now fully functional on Windows with a **4.5x speedup** over the initial unoptimized state (~31s/step vs 140s/step).

## 2. Technical Architecture

### The Problem: MSLK Dependency
In `torchao 0.16.0`, the `Int4WeightOnlyConfig` (version 2) uses a `mslk` backend for optimized kernels. On Windows, this backend is missing, and `torchao` fails to provide a performant alternative, often crashing or stalling.

### The Solution: `WindowsInt4Linear`
We bypassed the `torchao.quantize_` subclassing mechanism and implemented a native `nn.Module` replacement.

#### Key Components:
- **Storage**: Weights are stored as `uint8` tensors where each byte contains two 4-bit weights (High-4 and Low-4).
- **Scales & Zeros**: Stored as standard floating-point tensors, supporting asymmetric groupwise quantization (default `group_size=128`).
- **Dequantization Logic**: 
  - Unpacks the bits using bitwise operations (`>> 4` and `& 0x0F`).
  - Reconstructs the weight matrix on-the-fly during the `forward` pass.
  - Formula: `w = wq * scale + zero_point`
- **Compiler Optimization**: The `forward` pass is written using functional PyTorch operators (`torch.stack`, `F.linear`), which allow `torch.compile` (Inductor) to fuse the dequantization directly into the matrix multiplication kernel.

## 4. Configuration and Usage

To enable this mode, ensure your `.env` file or environment variables are set as follows:

```bash
# Set quantization mode to 4-bit
ACESTEP_QUANTIZATION_MODE=int4_weight_only

# Recommended: ensure compilation is enabled for maximum speed
ACESTEP_COMPILE_MODEL=true
```

The system will automatically detect the Windows environment and apply the `WindowsInt4Linear` replacement during model initialization.
